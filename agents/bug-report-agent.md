# Bug Report Agent

## Overview

The Bug Report Agent is responsible for generating structured bug reports when test failures are detected.

This agent collects failure information and converts it into a detailed bug report that can be shared with the development team.

---

## Responsibilities

The Bug Report Agent performs the following tasks:

* Analyze failed test cases
* Generate bug descriptions
* Document steps to reproduce
* Define expected and actual results
* Assign severity levels

---

## Bug Report Structure

A typical bug report generated by the agent includes:

* Bug Title
* Linear Issue URL
* Steps to Reproduce
* Expected Result
* Actual Result
* Severity Level
* Bug Screenshots

---

## Linear Integration

The Bug Report Agent integrates with Linear to create and manage bugs directly in the project's Linear board.

### Creating Bugs in Linear

* Use the Linear API to create new issues programmatically
* Map bug report fields to Linear issue properties (title, description, priority)
* Set appropriate issue types and status based on severity

### Attaching Screenshots

* Capture and attach bug screenshots to the Linear issue as evidence
* Upload screenshots to the Linear issue using the Linear API
* Include relevant visual context for UI-related bugs

### Labels and Project Assignment

* Apply appropriate labels to categorize the bug (e.g., "bug", "ui", "api", "critical")
* Assign the issue to the relevant project within the Linear workspace
* Set priority levels corresponding to bug severity (P0-P4)
* Add team assignments if applicable

### Linear Issue URL

Each bug report includes a direct link to the Linear issue for tracking and collaboration.

---

## Input

Failed test case details from the Result Analyzer Agent.

---

## Output

A structured bug report that can be used by developers to investigate and fix the issue.
